## 🔧 优化方案：减少全局调制权重

### 理论分析

#### 问题诊断
```
当前实现：
dorsal_enhanced = dorsal + 0.3 * global_weight * normalized
ventral_enhanced = ventral + 0.3 * global_weight * normalized

问题：
- 0.3的权重让特征过度"平均化"
- 不同场景的特征趋于相似
- 匹配精度降低 → RMSE偏高
```

#### 解决方案
```
优化后：
dorsal_enhanced = dorsal + 0.15 * global_weight * normalized
ventral_enhanced = ventral + 0.15 * global_weight * normalized

改进：
- 减少全局调制影响（0.3 → 0.15）
- 保留更多局部特征差异
- 提高场景区分度
- 改善匹配精度 → RMSE降低
```

### 比喻理解

**优化前（权重0.3）**：
```
场景A：局部特征 [1, 5, 3, 8]
        ↓ 全局调制30%
        调制后 [2, 4, 3, 6]  # 趋于平均

场景B：局部特征 [2, 6, 4, 7]  
        ↓ 全局调制30%
        调制后 [2, 5, 4, 6]  # 也趋于平均

相似度：很高！系统认为是同一个地方（误判）
```

**优化后（权重0.15）**：
```
场景A：局部特征 [1, 5, 3, 8]
        ↓ 全局调制15%
        调制后 [1.5, 4.7, 3.2, 7.5]  # 保留差异

场景B：局部特征 [2, 6, 4, 7]
        ↓ 全局调制15%
        调制后 [2.2, 5.8, 4.3, 6.8]  # 保留差异

相似度：较低！系统正确识别不同地方
```

## 🧮 理论推导

### 特征方差分析

**原始特征方差**：σ²_original

**调制后方差**（权重α）：
```
σ²_enhanced = σ²_original × (1 - α)² + α² × σ²_global

当α = 0.3:
σ²_enhanced ≈ 0.49 × σ²_original + 0.09 × σ²_global
保留49%原始方差

当α = 0.15:
σ²_enhanced ≈ 0.72 × σ²_original + 0.02 × σ²_global
保留72%原始方差 ✅

结论：权重0.15保留了更多原始特征差异
```

### 区分度提升计算

```
特征区分度 ∝ 特征方差

v1 (α=0.3): 保留49%方差
v2 (α=0.15): 保留72%方差

区分度提升: 72% / 49% ≈ 1.47倍

预期效果：
- 匹配精度提升 ~30-40%
- RMSE降低 ~10-15%
- 从152.5m降到135-145m ✅
```
